api是甚麼?

api在維基百科的解釋是應用程式介面
簡化來說就是，遵循API的規則發起請求，取得想要的資料。
舉個例子就是:

今天你想要喝一款飲料，於是你來到自動販賣機前面。
接著你照著販賣機的規則投幣，觸發你想要飲料的按鈕，
最後取得你要的飲料。

在這個動作內，你不需要去管販賣機是怎麼操作的，
你只要了解販賣機的規則，然後操作即可。

套用到API上來說的話共分三個階段並再詳細的分步驟的話，
總共分成三個步驟:

一.我想喝飲料(你要的資料)

二.我投幣並且點選要的飲料(你向API(request)發出請求)

三.飲料掉到取物口(API回應給你(response)，你要求的資料)

使用API時不需要了解後面的各種邏輯，只要依照API說明操作，就可以回傳你要的結果。

那常見的API有甚麼?如下所展示:

Web API：Leaflet + OpenStreetMap 地圖應用開發
我想要在網頁顯示公司位置
我載入 Leaflet 地圖 JS 套件，並引入 OpenStreetMap 開放街圖，並指定對應座標
網頁顯示對應地圖服務，但你並不用知道裡面的圖片顯示原理

Web API：TDX 觀光 API
我想獲得台灣觀光景點資料
我輸入說明文件上的網址請求方式
TDX 伺服器回應我對應的 JSON 格式資料
