檔名副檔名分割方法

當使用file上傳檔案時。我們可能會需要將檔名及副檔名分割並做出對應的動作。在這邊會提供兩個分割的方法供各位參考。

兩個方法的作用一樣，作法其實也相似，差別在於使用不同的API

第一種:pop()與split( )的搭配:

    let i = 'test.text.docx';?//此次所分割的字串

    let name = i.name.split('.');
    //將字串中以"."為界線進行分割並組成字串
    //此時的name = ['test','text','docx'];

    let type = name[name.length - 1];
    //抓出副檔名
    //因為陣列最後一個必然是副檔名，所以我們索引為最後一個值，也就是副檔名的字串docx。
        
    name.pop();//移除副檔名
    //接著我們將副檔名的索引移除
    //pop的功能就是將最後一個索引移除

    name = name.join('.');
    //最後將剩餘的字串再以原先為界線分割的'.'再次組合起來為字串
    //join的功用就是將陣列中的值，以('')使用者指定的字串組合起來
    //拿範例來說明就是將剩餘的test,text間加入.並轉為字串
    //所以最後的name = 'test.text'

第二種:substring( )與split( )的搭配:

    let i = 'test.text.docx';?//此次所分割的字串

    let name = i.name.split('.');
    //將字串中以"."為界線進行分割並組成字串
    //此時的name = ['test','text','docx'];

    let type = name[name.length - 1];
    //抓出副檔名
    //因為陣列最後一個必然是副檔名，所以我們索引為最後一個值，也就是副檔名的字串docx。

    //substring的型態:(開始位置,擷取數量)，用途是擷取指定字串並回傳。
    //例如字串為apple使用substring(0,3)。結果會是app。
    //所以上面的程式碼擷取了字串總數-5的字串。
    //name = 'test.text'
    //而之所以會-5是因為我們不擷取副檔名及含.的部分

整體來說會是POP的方法較為簡潔，因為副檔名的種類繁多，字數並非都是固定的。若使用第二種方法substring的話，會需要再寫判斷式來進行判斷。
